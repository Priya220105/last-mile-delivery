 🚀 QUICK START (5 MINUTES)
═══════════════════════════════════════════════════════════════════════════

1. SETUP MONGODB
   - Go to https://www.mongodb.com/cloud/atlas
   - Create free cluster (M0 free tier)
   - Copy your connection string

2. BACKEND SETUP
   cd backend
   npm install
   # Create .env file and paste MongoDB URL
   node init-db.js

3. FRONTEND SETUP
   cd ../frontend
   npm install

4. RUN LOCALLY
   # Terminal 1:
   cd backend && npm start
   
   # Terminal 2:
   cd frontend && npm run dev
   
   # Open http://localhost:3000

5. LOGIN WITH DEMO ACCOUNTS
   Customer: customer@example.com / password123
   Admin: admin@example.com / password123
   Agent: agent@example.com / password123

 📞 SUPPORT
═══════════════════════════════════════════════════════════════════════════

If something doesn't work:

1. Check SETUP_GUIDE.md Troubleshooting section
2. Verify MongoDB URL in .env
3. Check that both servers are running
4. Check browser console (F12) for errors
5. Check server logs in terminal

Common issues:
   - Port already in use → Change PORT in .env
   - MongoDB not connecting → Check URL and IP whitelist
   - Frontend not calling backend → Check proxy in vite.config.js
